Explanation:

What is Reinforcement Learning?

Reinforcement Learning (RL) is a type of Machine Learning. It is simply teaching a computer how to do something based on experiences and rewards. Think of a human experience: a child sees and touches something bright-red and gets a small burn. From this experience they learn that bright-red objects can sometimes be hot. This is an experience with a chosen action, a subsequent outcome, and a reward (in this case, a punishment).

In RL, a computer, or more specifically: an agent, can learn by leveraging probability. If touching a bright red object leads to a painful burn 75% of the time, then the agent is less likely to do that.

How does the agent learn?

The agent learns by utilizing a mathematical framework known as the Markov Decision Process. The agent, specific to this project, is trained in an environment that simulates rain and irrigation effects. From this environment, it learns that the chance of rain affects soil moisture. If there is a higher chance of the rain, the plants will need watered less or maybe not at all. If there is a lower chance of rain, the plants will likely need watered. It also learns that the action it chooses impacts soil moisture. Action 0 doesn't impact soil moisture at all, but the other actions do have varying impacts.

The target soil moisture for the agent is between 40% and 60%. The agent learns that anything outside this moisture range is not optimal. It knows the optimal range because every time an action is decided, the agent is rewarded or punished. If the agent waters the plants and the soil moisture is not optimal, it will be punished by losing points. If the agent correctly waters the plants, it is rewarded points.

The agent will begin to accrue experiences from this reward experience. These experiences are stored as probability tables. As more experiences are acquired, these probabilities will continue to change.

What can the agent do?

The moisture can be altered using 5 different actions. The first action is not doing anything, Action 0. All other actions are increasing variations of water to distribute, with Action 1 being a little water and Action 4 being a lot of water.

Looking at the graph below, it can be determined that the agent does not prefer Action 2 (green dot; light water) in any scenario. Also, Action 0 (blacks dot; no water) is highly favored once the soil moisture goes above 60%. Conversely, Action 4 (red dot; heavy water) is highly favored when the soil moisture goes below 20%.

The agent takes in the chance of rain via OpenWeather API. It takes in soil moisture via capacitive probes inserted at the base of the plants. Referring to its previous experiences learned from the training environment, the agent decides what action it should take. This action comes with a confidence factor.

In certain situations, the agent is likely to be more confident than others. If the soil moisture is very low and there is no chance of rain, the agent will likely be very confident in its decision to water.
